# atividade-scrum-Nicolas-Lyu-Inafuku



**Nome do projeto:**

Sistema de rastreamento de celulares



**Descrição:**

Meu  sistema tem  o objetivo de rastrear celulares perdidos através de uma solução tecnológica desenvolvida para localizar e monitorar dispositivos móveis em tempo real, usando recursos como: *GPS, redes móveis e conexão à internet.* Seu principal objetivo é fornecer informações precisas sobre a localização do aparelho, permitindo maior segurança, controle e recuperação em casos de perda ou roubo.

**Eventos do scrum:**

**Sprint:** São tarefas realizadas em um certo tempo, geralmente de 2 semanas, essas tarefas podem ser: *Login, catálogo, GPS...*



**Sprint Planning:** é o evento inicial do Scrum onde a equipe faz uma reunião para definir o objetivo da Sprint.



**Daily Scrum:** É uma reunião com objetivo é aumentar a colaboração, identificar obstáculos e sincronizar o trabalho do time.



**Sprint Review:** É um evento do Scrum realizado ao final da Sprint para inspecionar o incremento de produto.



**Sprint Retrospective:** é um evento Scrum focado na melhoria contínua, onde a equipe inspeciona o sprint anterior  e define melhorias práticas para o próximo sprint.





**Artefatos do Scrum:**

**Product Backlog:** é uma lista ordenada e dinâmica de tudo o que é necessário para criar e melhorar um produto.



**Sprint Backlog:** São itens do Backlog selecionados para serem feitos em uma sprint.



**Incremento:** é a soma de todos os itens do Product Backlog concluídos durante uma Sprint.





**Product Backlog:**

* Login
* GPS
* rastreadores
* Histórico de Rotas
* Status do Celular







**Prioridade do backlog:**

* 1-Login
* 2-GPS
* 3-rastreadores
* 4-Histórico de Rotas
* 5-Status do Celular



**Explicação:** O login é necessário para a segurança das informações do usuário, GPS é necessário já que ele será a ferramenta mais utilizada no sistema, os rastreadores serão necessários para a localização dos dispositivos, já o histórico e o status são funções mais secundárias.





**Sprint Planning:**

**Duração:** 3 Semanas

**3 itens do backlog:** Login, GPS, Histórico de rotas

**Sprint Backlog:**

**Login:** Login é necessário para o início de nosso sistema

**GPS:** GPS vai ser a ferramenta mais utilizada no sistema

**Histórico de rotas:** O histórico vai ser necessário para saber onde o dispositivo passou, caso a localização em tempo real tenha falhado.





**Burndown:**

**gráfico Burndown:** é uma ferramenta visual que mostra o trabalho restante em um dos eixos e dia a dia em outro eixo, dentro de uma Sprint, comparando com o plano com o que foi feito.



**Para que ele serve:** Ele serve para garantir que a equipe saiba se está no caminho certo para atingir a meta da Sprint.
